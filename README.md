## Asset state

The hero and service-card imagery is currently rendered as concept gradients
+ grid pattern. Asset prompts are saved to `prompts/_manifest.json`. To
generate real imagery later:

1. Set `GEMINI_API_KEY` in environment.
2. Run the prompts in `prompts/_manifest.json` against Gemini Nano Banana.
3. Drop the resulting JPGs into `public/hero/` and update the Hero component to
   reference them.

## Backend / admin

This run did NOT provision a Supabase backend. `auth_strategy = none` and the
intake doesn't include `SUPABASE_MANAGEMENT_PAT`, so Step 4.5 was skipped per
SKILL.md guidance. The reservation form and contact form are scaffolded as
client-component stubs — wire to email (Resend) or a CRM endpoint before
launch.
